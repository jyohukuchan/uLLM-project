uLLM-projectで使用される用語とその意味について
用語:意味
bpp:bit/parameter
BF16-error:BF16のraw-valueまたは出力を基準にした誤差
decode-tps:decode token/s
prefill-tps:prefill token/s
fq:推論エンジン上の実測を見て仕様を決める量子化またはformat候補の仮称。意味が確定したらこの定義を更新する
raw-value:scale・codebookを一切考慮しない各パラメータの値
codebook:LUTによる変換表
codebook-index:codebookを参照するために保存されるindex値
codebook-scope:同じcodebookを共有するtensor集合。familyより細かい機械的な分割にも使う
codebook-selector:blockごとに複数codebookのどれを使うか選ぶための追加metadata
scale table:scale-formatから生成され、scale-indexを実scale値へ変換する表
scale-index:scale tableを参照するために保存されるindex値
LUT:実装上の表参照機構。量子化仕様上の変換表はcodebookと書く
payload:FP8やFP16など、演算器または後段処理へ渡すための実際の保存・受け渡し値
logical payload bytes:tensorごとに参照されるpayload byte数の合計。共有payloadがある場合は重複分も数える
resident payload bytes:runtime buffer上に実際に保持されるpayload byte数の合計。共有payloadがある場合は重複分を数えない
payload role:.ullm.d内の参照payloadを用途別に選ぶ分類。tensor-index、tensor-scale、tensor-codebook、codebook、passthroughなど
tensor payload bundle:1つのquantized tensorに対応するindex、scale、codebookのpayload集合
passthrough payload bundle:1つのpassthrough tensorに対応するraw payloadとdtype、shape、elementsなどのmetadata集合
weight registry:runtime bufferへload済みのweight payloadをtensor単位で保持し、後続kernelから参照できるようにするregistry
resident payload:runtime buffer上に保持され、後続処理から再利用できるpayload
codebook pool:複数tensorから参照されるcodebook payloadを一度だけloadし、runtime bufferを共有するためのregistry内管理領域
loaded package handle:package summary、load済みweight registry、registry index一覧をまとめて保持するruntime側のpackage handle
chunked load:大きいpayloadを一定byte数ごとに読み、runtime bufferへ順番に転送するload方式
fused dequant:GEMM/GEMVなどの演算kernel内でcodebook-indexやscaleを実効値へ変換し、そのまま演算へ使う処理
matvec:行列とvectorの積。初期runtimeではmaterialize済みf32 weightとf32 inputを使う最小kernel境界として扱う
materialize:変換後のpayloadや実効値を別bufferへ一度書き出す処理
materialize smoke:runtime buffer上のquantized payloadをf32などへ展開し、後続kernel境界との接続を確認するsmoke
materialize matvec smoke:quantized payloadをf32へmaterializeした後、同じruntime上でmatvecまで実行して後続演算境界との接続を確認するsmoke
materialize bench:materialize処理をwarmup後に複数回実行し、平均・最小・p50・p95時間を測るbenchmark入口
staging fallback:backend bufferを一度host memoryへ移してCPU処理し、結果をbackend bufferへ戻す一時的な正しさ確認用fallback。性能評価用の実装とは扱わない
block:同じlocal-scaleを共有するraw-valueのまとまり
block-size:1つのblockに含まれるraw-valueの数
KV cache:attention decode用のkey/value cache
block table:sequenceが参照するKV cache block idの配列
group:実装上のblockの旧称または変数名。文書ではblockを優先する
superblock:複数のblockをまとめた補正metadataの共有単位
superblock補正:superblock単位に保存する追加metadataでlocal-scaleや実効値を補正する方式
outlier補正:superblock内の大きな残差を少数保存し、dequant後に加算して局所的な誤差を減らす方式
local-scale:数十個のraw-valueごとのscale
unsigned local-scale:符号bitを持たず、常に正値として扱うlocal-scale
signed local-scale:符号bitを持ち、blockごとに正負を選べるlocal-scale。診断・比較用語として使い、仕様候補として扱う場合は明記する
UEaMb:unsigned local-scale用のE/M形式。aはexp bit数、bはmantissa bit数を表す。例: UE4M3、UE5M3
monotonic floor:上位scale形式を評価するとき、下位scale形式の既存解を上位scale tableへ写像してbest-so-farに入れる下限保証
tensor-scale:1つのtensor全体にかかるscale
family-scale:同じfamilyに属するtensorで共有または補正に使うscale
global-scale:local-scaleより大きい単位でのscale。tensor-scaleやfamily-scaleを含む上位概念として使う
family:Q/K/V/OやMLPなど、性質が近いtensorをまとめる分類
control plane:request管理、scheduler、telemetry、APIなど、kernel実行そのもの以外を制御する層
runtime:weight、KV cache、backend、kernel呼び出しを管理してmodel executionを実行する層
runtime buffer:runtimeがbackend上またはCPU fallback上に確保するbyte buffer
runtime stream:backend上の非同期copyやkernel実行の順序を束ねる実行stream
host-device-copy:host memoryとruntime bufferの間でpayloadを転送する処理
RMSNorm:root mean square normalization。Qwen系decoder blockでlinear/attention前に使う正規化処理
RMSNorm smoke:runtime buffer上のf32 inputとweightからRMSNormを実行し、CPU/HIP kernel境界を確認するsmoke
package RMSNorm smoke:.ullm.d内の実際のpassthrough RMSNorm weightをf32へ変換し、runtime buffer上の決定的inputと組み合わせてRMSNorm kernel境界を確認するsmoke
SiLU-mul:MLPのgate/up中間値に使うelementwise処理。`silu(gate) * up` を計算する
SiLU-mul smoke:runtime buffer上のf32 gate/upからSiLU-mulを実行し、CPU/HIP kernel境界を確認するsmoke
Sigmoid-mul:Qwen3.5 self-attentionのoutput gateなどに使うelementwise処理。`sigmoid(gate) * input` を計算する
Sigmoid-mul smoke:runtime buffer上のf32 gate/inputからSigmoid-mulを実行し、CPU/HIP kernel境界を確認するsmoke
runtime add smoke:runtime buffer上の2つのf32 tensorを要素ごとに加算し、residual addなどで使うelementwise境界をCPU/HIP kernelで確認するsmoke
runtime RoPE smoke:runtime buffer上のf32 sequence/head tensorへsplit-half RoPEを適用し、CPU/HIP kernel境界を確認するsmoke
runtime causal attention smoke:runtime buffer上のf32 q/k/vからGQA対応のcausal softmax attention value mixを実行し、CPU/HIP kernel境界を確認するsmoke
runtime decode attention smoke:runtime buffer上の1-token f32 qとprefill済みf32 K/V cacheからGQA対応のsoftmax attention value mixを実行し、decode時のKV cache参照境界をCPU/HIP kernelで確認するsmoke。block tableやpaged KV cacheはまだ含まない
runtime paged decode attention smoke:runtime buffer上の1-token f32 q、物理block配置のf32 K/V cache、u32 block tableからGQA対応のdecode attentionを実行し、paged KV cache参照境界をCPU/HIP kernelで確認するsmoke。allocatorやscheduler全体はまだ含まない
runtime paged KV write smoke:runtime buffer上の1-token f32 K/Vとu32 block tableから、指定cache positionに対応する物理paged K/V cache slotへK/Vを書き込む境界をCPU/HIP kernelで確認するsmoke
paged decode state:runtime buffer上のblock table、paged K/V cache、1-token K/V scratch、q/output scratch、written_lenをまとめて保持し、runtime paged KV writeとruntime paged decode attentionを接続するdecoder step用state。cache readbackは論理順ではなく物理cache layoutを返す
paged decode step:1 token分のq/k/vを受け取り、k/vをpaged K/V cacheへ書き込んだ後、その時点のprefix長でpaged decode attentionを実行するdecoder step
Qwen3 self attention decode state:Qwen3/Qwen3.5系の狭いself-attention decode用状態。PagedDecodeStateを使い、softmax scaleを保持したうえで1 token分のq/k/vからattention outputを返す
Qwen3 self attention block step state:Qwen3/Qwen3.5系の狭いself-attention block step用状態。paged K/V write、paged decode attention、Qwen3.5 output gate、o projection、residual addを1 token分ずつ接続する。post RMSNormやMLPは含まない
runtime KV paged decode smoke:Rust側のKvBlockAllocatorで非連続block tableを作り、そのblock tableとhost側で検証用にpackした物理paged K/V cacheをruntime paged decode attentionへ渡すsmoke。scheduler管理とruntime kernel境界の最小接続を確認する
depthwise conv1d:channelごとに独立した1次元畳み込み。linear attentionのconv1d.weightでは入力系列の各channelに同じchannelのkernelだけを適用する
runtime depthwise conv1d smoke:runtime buffer上のf32系列とf32 weightからcausal depthwise conv1dを実行し、CPU/HIP kernel境界を確認するsmoke
MLP smoke:RMSNorm、gate/up/down matvec、SiLU-mulを小さい固定tensorで接続し、decoder MLP相当のruntime workflowを確認するsmoke
package MLP smoke:.ullm.d内の実際のgate/up/down quantized tensorをmaterializeし、matvecとSiLU-mulでMLP相当のruntime workflowを確認するsmoke
package RMSNorm MLP smoke:.ullm.d内の実際のpassthrough RMSNorm weightとgate/up/down quantized tensorを使い、RMSNormからMLPまでをruntime上で接続するsmoke
package MLP block smoke:.ullm.d内の実際のpost_attention_layernorm.weightとMLP quantized tensorを使い、RMSNormからMLP workflow、residual addまでを接続するsmoke。attentionやdecoder block全体の計算ではない
package linear attention projection smoke:.ullm.d内の実際のlinear_attn projection quantized tensorをmaterializeし、共通の決定的inputからmatvecまで接続するsmoke。linear attention全体の計算ではない
package self attention projection smoke:.ullm.d内の実際のself_attn q/k/v/o projection quantized tensorをmaterializeし、共通の決定的inputからmatvecまで接続するsmoke。通常self-attention全体の計算ではない
package self attention QK norm smoke:.ullm.d内の実際のself_attn q/k projection quantized tensorとq_norm/k_norm passthrough tensorを使い、projection出力をhead単位でRMSNormへ接続するsmoke。RoPE、attention score、KV cacheはまだ含まない
package self attention RoPE smoke:.ullm.d内の実際のself_attn q/k projection quantized tensorとq_norm/k_norm passthrough tensorを使い、projection、head単位RMSNorm、runtime RoPEまで接続するsmoke。attention score、KV cacheはまだ含まない
package self attention attention smoke:.ullm.d内の実際のself_attn q/k/v projection quantized tensorとq_norm/k_norm passthrough tensorを使い、projection、head単位RMSNorm、runtime RoPE、runtime causal attention value mixまで接続するsmoke。Qwen3.5のgated q projectionではper-headのquery/gate splitを行い、queryだけをattentionへ渡す。o_proj、residual、KV cacheはまだ含まない
package self attention decode smoke:.ullm.d内の実際のself_attn q/k/v projection quantized tensorとq_norm/k_norm passthrough tensorを使い、prefill済みK/V cache相当の系列と最後の1 token queryをruntime decode attentionへ渡すsmoke。full causal attentionの最後のtokenと比較し、同じK/V系列をKvBlockAllocator由来のblock tableに従ってruntime paged KV writeで物理paged K/V cacheへ書き込み、そのruntime bufferをruntime paged decode attentionへ渡す経路も検証する。scheduler全体はまだ含まない
package self attention block smoke:.ullm.d内の実際のself_attn q/k/v/o projection quantized tensorとq_norm/k_norm passthrough tensorを使い、projection、head単位RMSNorm、runtime RoPE、runtime causal attention value mix、runtime Sigmoid-mulによるQwen3.5 output gate、o projection、residual addまで接続するsmoke。MLP、KV cacheはまだ含まない
package self attention MLP block smoke:.ullm.d内の実際のself_attn q/k/v/o projection quantized tensor、q_norm/k_norm、post_attention_layernorm.weight、MLP tensorを使い、Qwen3.5 self-attention block出力をpost RMSNorm、MLP、final residual addまで接続するlayer-level partial decoder smoke。self-attention側はfull causal attentionを参照として残しつつ、Qwen3 self attention block step stateでruntime paged KV write、paged decode attention、output gate、o projection、residual addを1 tokenずつ実行する
package linear attention QKV norm smoke:.ullm.d内の実際のlinear_attn.in_proj_qkv quantized tensorとlinear_attn.norm.weight passthrough tensorを使い、QKV projection出力の先頭128要素にRMSNormを適用するsmoke。linear attention全体の計算ではない
package linear attention conv1d smoke:.ullm.d内の実際のlinear_attn.in_proj_qkv quantized tensorから複数timestepのQKV出力を作り、linear_attn.conv1d.weight passthrough tensorでcausal depthwise conv1dを適用するsmoke。linear attention全体の計算ではない
package linear attention recurrent smoke:.ullm.d内の実際のlinear_attn.in_proj_qkv / conv1d / in_proj_a,b / A_log / dt_biasを使い、q/k/v split と gate/beta を recurrent境界へ接続するsmoke。linear attention全体の計算ではない
package linear attention post smoke:.ullm.d内の実際のlinear_attn.in_proj_z / norm / out_projを使い、post-recurrentのgated RMSNormからout projectionまでを接続するsmoke。linear attention全体の計算ではない
package linear attention block smoke:.ullm.d内の実際のinput_layernorm.weightとlinear_attn系tensorを使い、input RMSNormからlinear attention workflow、residual addまでを接続するsmoke。MLPやdecoder block全体の計算ではない
package linear attention workflow smoke:.ullm.d内の実際のlinear_attn.in_proj_qkv / conv1d / in_proj_a,b / A_log / dt_bias / in_proj_z / norm / out_projを使い、recurrent境界からpost-recurrent gated RMSNormとout projectionまでを接続するsmoke。linear attention全体の計算に近いが、まだ通常decoder layer全体ではない
package linear attention auxiliary smoke:.ullm.d内の実際のlinear_attn補助passthrough tensorをf32へ変換し、runtime bufferへの転送とreadbackを検証するsmoke。linear attention全体の計算ではない
package linear attention MLP block smoke:.ullm.d内の実際のinput_layernorm.weight、linear_attn系tensor、post_attention_layernorm.weight、MLP tensorを使い、linear attention block出力をMLP blockのresidual inputに渡すlayer-level partial decoder smoke。既定は1 tokenで、任意のsequence長では各tokenのMLPまで接続する。通常attention層やKV cache管理はまだ含まない
linear attention gate beta:linear attentionでin_proj_a/bの出力、A_log、dt_biasからgateとbetaを作る境界処理
runtime linear attention gate beta smoke:runtime上でa/b、A_log、dt_biasからgate/betaを計算し、決定的入力で境界動作を確認するsmoke。linear attention全体の計算ではない
package linear attention gate beta smoke:.ullm.d内の実際のlinear_attn.in_proj_a/b quantized tensorとA_log/dt_bias passthrough tensorを使い、runtimeでgate/beta境界を検証するsmoke。linear attention全体の計算ではない
linear attention recurrent:linear attentionの逐次state更新処理。uLLMではq/kをkey head layout、v/gate/beta/output/stateをvalue head layoutで扱う
runtime linear attention recurrent smoke:runtime上で小型f32 q/k/v/gate/beta/stateからlinear attention recurrent境界を計算し、更新後stateとoutputを検証するsmoke。linear attention全体の計算ではない
HIPRTC:HIP runtime compilation。通常C++でbuildしたruntimeからHIP kernel sourceを実行時compileしてmodule launchする仕組み
JIT kernel:実行時にcompileしてloadするkernel。uLLMではHIPRTC経由のAQ4 materialize kernelなどを指す
kernel-required smoke:staging fallbackを禁止し、実kernel経路だけで成功することを確認するsmoke
backend:HIP、CPU、CUDA、CANN、JAX/TPUなど、runtimeが実際に演算へ使う実装単位
backend capability:backendごとに異なる対応機能や推奨値。block-size、dtype、kernel variantなどを含む
telemetry:実行中に記録する性能・メモリ・allocator状態などの測定値
